# Information Architecture And Sections

## Page Model

The website is a single-page scrolling academic project homepage. It opens on the Home section and uses fixed navigation for live presentation movement.

## Global Elements

### Institutional Identity

Fixed top-left identity cluster:

- HUST
- Universite Paris-Saclay / Evry

The current prototype uses text-based placeholders. Later versions should replace these with official logo assets and link to institutional homepages.

### Navigation

Fixed top-right trigger:

- Default state: right-pointing triangular icon
- Active state: rotated 90 degrees clockwise, pointing downward
- Opens a floating dropdown menu
- Menu items scroll smoothly to section anchors

## Section Responsibilities

### 1. Home

Responsibilities:

- Establish project identity
- Show PPEI_B3_S2, Team B, Group 4
- Present the project as an academic homepage, not a commercial landing page
- Give a compact preview of the knowledge system

### 2. Team Stuff

Responsibilities:

- Present four team members equally
- Show headshot placeholder, name, basic info, jobs of interest, and fields of activity
- Support later replacement with real portraits and member content

### 3. Gantt Chart Of Team Work Management

Responsibilities:

- Represent project management as a web-native timeline
- Keep task ownership and week structure readable
- Serve as placeholder for the future real Gantt data

### 4. Database & Interactome

Responsibilities:

- Show the conceptual interactome as a node network
- Provide structured access to interview database content
- Avoid a dense raw table by using person switching and expandable categories

### 5. Resume / Summary + Main Advice

Responsibilities:

- Separate source interview summaries from final synthesized advice
- Use two columns for presentation clarity
- Emphasize final recommendations

### 6. Proof Of Professional Behavior

Responsibilities:

- Provide a prepared evidence area
- Support future screenshots, text evidence, meeting notes, and behavior records
- Avoid looking incomplete while content is still pending
